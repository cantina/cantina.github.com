app Class

Defined in: node_modules/cantina/lib/app.js:8
Module: cantina

Cantina application instance

Presuming you created your cantina application in 'http' mode (the default), the app instance is an http flatiron app. A ton of functionality is inherited from flatiron and cantina extends it with some more.

A simple cantina app is an http server that responds to a '/' route.

var cantina = require('cantina');

var app = cantina.createApp({
  name: 'my-app',
  description: 'My App',
  version: '0.1.0',
  amino: false, // Amino is for clustered apps, turn it off.
  host: 'localhost', // Optional, this is the default host.
  port: 8080 // Optional, this is the default port
});

app.router.get('/', function() {
  this.res.html('<h1>My App</h1><p>Welcome to my awsome app!</p>');
});

app.start();

Provided by the views module.

Defined in node_modules/cantina-views/lib/plugin.js:111

Render views.

This function is available in the 'router' scope (ie this in a route handler). It is NOT available on app.

Parameters:

  • view String

    The path to a template, relative to ANY registered views namespace, excluding the template extension.

  • opts Object

    Template data and/or engine-specific options.

  • [cb] Function optional

    (err, str) Callback to handle the rendered text. A default callback is provided which calls this.res.html(str).

Example:

var app = cantina.app;
app.use(require('cantina-views').plugin);

// The default cantina-views options assume you have a ./lib/views
// directory and registers it as the 'root' namespace.  Templates
// inside the registered directory can then be rendered like:

app.router.get('/', function() {
  this.render('index', {title: 'A title', body: 'The Body'})
});

use

  • plugin
  • [options]
)

Defined in node_modules/cantina/lib/app.js:39

Attach a plugin to the application. Plugins should conform to the following interface:

var plugin = {
  "name": "example-plugin", // Plugin's name

  "attach": function attach(options) {
    // Called with plugin options once plugin attached to application
    // this - is a reference to application
  },

  "detach": function detach() {
    // Called when plugin detached from application
    // (Only if plugin with same name was attached)
    // this - is a reference to application
  },

  "init": function init(callback) {
    // Called on application initialization
    // App#init(callback) will be called once every plugin will call callback
    // this - is a reference to application
  }
};

Parameters:

  • plugin Object

    The plugin to attach.

  • [options] Object optional

    Options to pass to he plugin durring attachment.
